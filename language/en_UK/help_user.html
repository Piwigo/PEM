Contributor's guide


Add an extension

To contribute to the project, you need first to create an extension. Click on Add an extension in the menu.

  1. Give it a name
  2. Choose one or more categories
  3. Fill in the description. Multilingual tags are available, so the visitor can read the content in his own language.

On the extension displaying page, a button set allows several actions.

  1. edit the extension data.
  2. delete the extension.
  3. manage external links
  4. add a picture to illustrate the extension
  5. add a revision
Add a revision

Each revision is an extension new version. Thus, there are several revisions for an extension. To create a new revision, click on the "add a revision" button on the extension displaying page to display a form:

  1. Number the revision, for example "2.0"
  2. Select the corresponding ZIP archive
  3. Choose a list of the extended software versions that are compatible with the extension revision. For example, if the extended software is named "Piwigo" and the extension is "dynamic menu", you should state that revision "2.0" of "dynamic menu" is compatible with versions 2.0.0, 2.0.1 and 2.0.2 of "Piwigo".
  4. List the changes brought by this new version. Multilingual tags are available, so the visitor can read the content in his own language.
Edit a revision
  1. Modify a revision
  2. Delete a revision
Mind out:
  • A revision archive cannot be modified. If there is an error in the archive, one must delete the revision and create a new one.
  • If the word about your contribution has already been spread, create a new version rather than suppress the contribution. Otherwise, the users could assume they have your last version though they are in fact using a former one.
Multilingual descriptions

Multilingual descriptions must be inserted between [lang=xx] and [/lang] tags, where xx is the language code (for example, fr, en, es, it, etc.) There are also 2 special tags: [lang=default] and [lang=all]
