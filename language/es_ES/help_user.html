Añadir una extensión

Para aportar tu extensión, primero debes crearla. Una vez lo hayas hecho, haz clic sobre Añadir una extensión en el menú.

  1. Dale un Nombre
  2. Elige una o varias categorías
  3. Introduce una descripción. Puedes utilizar etiquetas multilingües para que el contenido de la descripción se adapte al idioma del usuario.

En la página donde se visualiza la extensión, un conjunto de botones permite realizar varias acciones.

  1. Modificar la información de la extensión.
  2. Eliminar la extensión.
  3. Administrar vínculos externos
  4. Añadir una imagen que muestre la finalidad de la extensión
  5. Añadir una revisión
Añadir una revisión

Cada revisión corresponde a la evolución de una extensión, es decir, a una nueva versión. Puede haber varias revisiones por extensión en función de las mejoras o correciones que sufra la misma. Para crear una nueva revisión, clica sobre "Añadir una revisión" en la página de visualización de la extensión. Aparecerá el siguiente formulario:

  1. Numerar la revisión, por ejemplo "2.0"
  2. Selecciona el archivo correspondiente (*.zip)
  3. Selecciona todas las versiones que serán compatibles con esta revisión de la extensión. Por ejemplo, si el software extendido se llama "Piwigo" y la extensión "menú dinámico", entonces hay que declarar aquí que la revisión "2.0" de "menú dinámico" es compatible con las versiones 2.0.0, 2.0.1 y 2.0.2 de "Piwigo".
  4. Indicar el registro de cambios realizados para esta nueva revisión. Es posible utilizar las etiquetas multilingües para que el contenido se adapte al idioma elegido por el usuario.
Modificar una revisión
  1. Editar revisión
  2. Eliminar revisión
Atención:
  • El archivo de una revisión no puede ser modificado. Si hubiera algún error en el archivo, debe eliminarse esa revisión y crear una nueva.
  • En caso de que ya se hubiera difundido la revisión errónea, lo mejor es crear una nueva revisión para que los usuarios puedar darse cuenta que existe una nueva actualización disponible para su extensión. Es preferible eso que borrarla, para así evitar que los usuarios usen una revisión obsoleta.
Descripciones multilingües

Las descripciones multilingües se introducen entre las etiquetas [lang=xx] y [/lang], donde xx es el código del idioma (por ejemplo fr, en, es, it...) Existen además dos etiquetas especiales: [lang=default] y [lang=all]
