Guide du contributeur


Ajouter une extension

Pour contribuer au projet, la première étape consiste à créer une extension. Cliquer sur Ajouter une extension dans le menu.

  1. Donner un nom
  2. Choisir une ou plusieurs catégories
  3. Remplir la description. Il est possible d'utiliser les balises multilingues pour que le contenu s'adapte à la langue choisie par le visiteur.

Sur la page de visualisation de l'extension, une barre de boutons donne accès à plusieurs actions.

  1. modifier les informations de l'extension.
  2. supprimer l'extension.
  3. gérer des liens externes
  4. illustrer l'extension avec une image
  5. ajouter une révision
Ajouter une révision

Chaque révision correspond à l'évolution d'une extension. Il y a donc plusieurs révisions par extension. Pour créer une nouvelle révision, cliquer sur le bouton "ajouter une révision" de la page de visualisation de l'extension. Un formulaire apparaît :

  1. Numéroter la révision, par exemple "2.0"
  2. Sélectionner l’archive correspondant, au format ZIP
  3. Choisir une liste de versions du logiciel étendu qui seront compatibles avec cette révision de l'extension. Par exemple, si le logiciel étendu se nomme "Piwigo" et l'extension "dynamic menu", alors il faut déclarer ici que la révision "2.0" de "dynamic menu" est compatible avec les versions 2.0.0, 2.0.1 et 2.0.2 de "Piwigo".
  4. Indiquer la liste des changements apportés par cette nouvelle révision. Il est possible d'utiliser les balises multilingues pour que le contenu s'adapte à la langue choisie par le visiteur.
Modifier une révision
  1. Editer la révision
  2. Supprimer la révision
Attention :
  • L’archive d’une révision ne peut pas être modifiée. En cas d’erreur dans l’archive, il faut supprimer la révision et en créer une nouvelle.
  • Dès que vous avez communiqué à propos de votre contribution ne la supprimez pas. Créez plutôt une nouvelle révision. En effet, les utilisateurs de votre contribution pourraient croire qu'ils utilisent votre dernière révision alors qu’ils sont encore sur une version précédente.
Descriptions multilingues

Les descriptions multilingues se font entre les balises [lang=xx] et [/lang], ou xx est le code de la langue (par exemple fr, en, es, it, etc....) Il existe également 2 balises spéciales: [lang=default] et [lang=all]

Exemple d'utilisation :

Si le contenu n'est pas disponible dans la langue choisie par le visiteur, c'est l'anglais, la langue par défaut de PEM, qui sera utilisé.
